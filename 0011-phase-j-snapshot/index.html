TIER 2 · PHASE J

快照 — get_as_of(snapshot) 时间旅行

H 让每个版本带上 seq。读路径处处用 snap = Seq::MAX(读最新)。Phase J 只做一件事:把那个写死的 snap 参数化。于是有了 MVCC 时间旅行——抓一个快照号,之后无论怎么写、怎么删,拿这个快照读到的永远是"抓快照那一刻的一致视图"。这就是数据库"可重复读"、"一致性备份"、"长事务读"的地基。本课聚焦点读(get_as_of);范围读的快照版(对外 scan 游标)是 Phase K。

几乎零新代码——这正是 H 设计对的回报。 get_newest(uk, snap) 在 H2 就带了 snap 参数;lookup_key(uk, snap) 在 H1 就支持任意 snap。J 只是不再恒传 MAX,而是传一个捕获的快照号。把"读哪个版本"做成一个参数,而非写死,是 J 能一行落地的全部秘密。

一个最小步

本阶段是单课:J — snapshot() + get_as_of。

  • Engine::snapshot() -> Seq:抓当前序列号(独占上界)。
  • get_as_of(key, snapshot):只看 seq < snapshot 的版本;get = get_as_of(key, self.seq)。
  • 顺手修一个 H1 潜伏的 lookup_key seek-type bug(snap=MAX 时被掩盖,有限快照才现形)。
承接 Phase K. J 把点读参数化成快照。K 把范围读也参数化:对外 scan 游标(seek/next 流式),按 snapshot 过滤版本 + 跳墓碑 + 按 user_key 去重 —— 即 RocksDB 的 DBIter。K 收尾后,整个引擎的读路径(点 + 范围)都是快照一致 + 墓碑感知的,M2 多版本引擎达成。
