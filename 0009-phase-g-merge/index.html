TIER 2 · PHASE G

读时归并 — MergeIterator + scan

Phase F 把每种数据源都装成同一个 LsmIterator。Phase G 组装它们:把 active memtable、若干 immutable、一摞 SST 这些各自有序的源,用一个 MergeIterator 归并成一条全局有序、新值遮旧值的流。这就是 LSM 的灵魂——读时归并(merge-on-read):写时只追加,读时才把分层的有序片段合一。

本阶段只做纯归并 + plain key。 同 key 谁新谁旧,这一阶段只能靠源优先级(active > immutable > 新 SST > 旧 SST)区分;比较器是裸字节 Ord、外置在 MergeIterator 里。到 Phase H3,seq 编进 key、比较器换成 InternalKeyComparator——届时只改"存的 key 字节 / merge 比较器 / 上层解码"三处,MergeIterator 的 next/堆逻辑一字不动。这次"比较器重写"是刻意的教学点(亲历 seq 如何改变去重语义)。快照与墓碑感知扫描留到 H/K。

两个最小步

STEP G1 · done

MergeIterator:min-heap k 路归并

merge.rs:MergeIterator<'a> 持 Vec<Box<dyn LsmIterator>> + min-heap BinaryHeap<Reverse<(Vec<u8>, idx)>>。看齐 N 源的头、取最小、只推那一个;同 key 多源按源优先级去重。第三次破自引用(堆存 key 副本而非借源)。对标 merging_iterator.cc。

harness:归并差分 vs 合并后 BTreeMap · 源优先级去重 · 跨源 seek
STEP G2 · done

minimal scan(start..end):对外范围扫描雏形

把引擎的 mem / imm[..] / ssts[..] 各做一个 LsmIterator,按优先级塞进 MergeIterator,seek(start) 起、吐到 key >= end 止。这是引擎对外的第一个范围读 API(plain key)。对标 db_iter.cc 雏形。

harness:engine.scan(a..b) == BTreeMap::range(a..b)(引擎级差分)
承接 Phase H. G 在 plain key 上跑通后,Phase H 引入 InternalKey = user_key + seq + type:写读路径全穿、H3 把 MergeIterator 的比较器换成 InternalKeyComparator(同 user_key 按 seq 降序),去重从"源优先级"升级成"seq 定序",并把墓碑(kTypeDeletion)折入读路径。G' 的 minimal scan 到 Phase K 升级成快照 + 墓碑感知的完整对外 scan。

本阶段交付的不变量

不变量怎么验步
归并差分N 源 full scan == 按优先级合并进 BTreeMap 的有序遍历(or_insert 不覆盖 = 高优先级赢)G1
源优先级去重同 key 多源 → 只吐最高优先级源的值,且只一次(旧值被遮蔽、推进跳过)G1
跨源 seekseek(t) 后落在所有源里第一个 ≥t 的 key(命中/缝隙/超界)G1
引擎级扫描差分engine.scan(a..b) 逐条 == BTreeMap::range(a..b)(穿真实 mem+imm+SST)G2
