KV 类型 + 比较器

整个 LSM 的地基是一个字：序。MemTable 有序、SST 有序、compaction 合并有序——全靠一个比较器定义"谁在前"。这一步只写它。

GOAL

交付 src/kv.rs：Key/Value 类型别名 + 一个 Comparator trait + 默认字节序实现 BytewiseComparator。

Done when: cargo test kv 全绿，表驱动用例覆盖：相等、前缀短于扩展、空串、逆序对。

为什么先做这个（mission 锚）

RocksDB 把比较器抽成可替换接口 Comparator，不是写死字节序——MyRocks 就靠自定义比较器把 SQL 索引的整数/字符串编码成可字节序比较的 key。你在 learning-record 0002 见过这种"RocksDB 是零件"的弹性。地基这一层留好接缝，后面才接得上。

代码 1 — src/kv.rs（核心）

// src/kv.rs — KV 基本类型与比较器（mini-LSM 的"序"之根）
use bytes::Bytes;
use std::cmp::Ordering;

// Bytes = 引用计数的不可变字节缓冲，clone 是 O(1) 浅拷贝。
// 对标 RocksDB 的 Slice（指针+长度，零拷贝传递 key/value）。
pub type Key = Bytes;
pub type Value = Bytes;

/// key 上的全序关系。对标 RocksDB 的 Comparator 接口。
/// 抽成 trait = 留接缝：将来可换成"数值序""带 timestamp 序"而不动 MemTable。
pub trait Comparator: Send + Sync {
    fn compare(&self, a: &[u8], b: &[u8]) -> Ordering;
    fn name(&self) -> &str;
}

/// 默认实现：原始字节的字典序。对标 BytewiseComparatorImpl。
pub struct BytewiseComparator;

impl Comparator for BytewiseComparator {
    fn compare(&self, a: &[u8], b: &[u8]) -> Ordering {
        a.cmp(b) // &[u8] 的 Ord 即逐字节字典序
    }
    fn name(&self) -> &str {
        "mini.BytewiseComparator" // 命名是为了将来格式兼容性校验（见核心问题 3）
    }
}

Harness 系统 ①：表驱动测试

为什么这里用表驱动

比较器的 bug 全在边界：空串、前缀关系、相等、对称性。这类"同一段断言逻辑 × 一堆输入用例"的场景，表驱动是最省的—— 用例表是数据，断言是代码，加一行用例零成本，且漏掉的边界一眼能看出来。这是 Go 标准库和 RocksDB 参数化测试的主力风格，纯可迁移技能。

代码 2 — 表驱动 harness（接在 kv.rs 末尾）

#[cfg(test)]
mod tests {
    use super::*;

    // 一行 = 一个用例：(左, 右, 期望的序关系, 这条在验什么)
    fn cmp(a: &[u8], b: &[u8]) -> Ordering {
        BytewiseComparator.compare(a, b)
    }

    #[test]
    fn bytewise_order_table() {
        let cases: &[(&[u8], &[u8], Ordering, &str)] = &[
            (b"a",   b"a",   Ordering::Equal,   "相等"),
            (b"a",   b"b",   Ordering::Less,    "逐字节小"),
            (b"a",   b"ab",  Ordering::Less,    "前缀短于扩展"),
            (b"ab",  b"a",   Ordering::Greater, "扩展长于前缀"),
            (b"",    b"a",   Ordering::Less,    "空串最小"),
            (b"",    b"",    Ordering::Equal,   "空=空"),
            (&[0xFF], &[0x00], Ordering::Greater, "按 u8 非符号比较"),
        ];
        for (a, b, want, why) in cases {
            assert_eq!(cmp(a, b), *want, "用例失败: {}", why);
            // 对称性不变量：a?b 必须与 b?a 互为反序
            assert_eq!(cmp(b, a), want.reverse(), "对称性破: {}", why);
        }
    }
}
