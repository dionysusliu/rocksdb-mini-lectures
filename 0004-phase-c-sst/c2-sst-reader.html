STEP C2 · Phase C SST

SstReader — 二分点查

把 C1 写出的字节读回来。关键收获:不必把整个文件载入逻辑结构——读个固定 footer 定位 index,二分索引,只去 data 区捞那一条 value。这是 SST 比"重放 WAL"快的根源。

GOAL

交付 SstReader:open(path)(读 footer + 载入 index)+ get(key)(二分 + 读 entry)。

Done when: 差分 harness——随机有序输入,SstReader::get 与 BTreeMap::get 逐键一致;边界键(小于全部 / 大于全部 / 不存在)返回 None。

读路径三步(对标 RocksDB)

① 从文件尾读 24B footer → 拿到 index_offset/index_len/magic;② 校验 magic、解析 index 进内存;③ get 对 index 二分,命中则按偏移去 data 区读那条 entry 的 value。

对标 table/block_based/block_based_table_reader.cc(Open 读 footer → 读 index block → Get 二分)

步骤

  1. 给 src/sst.rs 补「代码 1」SstReader(open + get)。注意顶部 use 补 std::io::Read。
  2. 把「代码 2」的两个测试加进 sst.rs 的 tests 模块(和 C1 的 rejects_unsorted 并列)。
  3. 跑命令看绿。

代码 1 — sst.rs 的 SstReader

// 顶部 use 区补:
use std::io::Read;

/// 只读打开一个 SST,支持点查。MVP:整文件入内存(RocksDB 按 block 读 + cache)。
pub struct SstReader {
    raw: Vec<u8>,
    index: Vec<(Bytes, u64)>, // 已排序的 (key, data偏移)
}
impl SstReader {
    pub fn open(path: &Path) -> Result<Self> {
        let mut raw = Vec::new();
        File::open(path)?.read_to_end(&mut raw)?;
        if raw.len() < FOOTER_LEN { bail!("SST 太小,无 footer"); }
        // ① 从尾部读 footer
        let ft = &raw[raw.len() - FOOTER_LEN..];
        let index_offset = u64::from_le_bytes(ft[0..8].try_into()?) as usize;
        let index_len = u64::from_le_bytes(ft[8..16].try_into()?) as usize;
        let magic = u64::from_le_bytes(ft[16..24].try_into()?);
        if magic != MAGIC { bail!("magic 不符,非本格式 SST"); } // ② 校验格式
        // ③ 解析 index 进内存
        let idx = &raw[index_offset..index_offset + index_len];
        let count = u32::from_le_bytes(idx[0..4].try_into()?) as usize;
        let mut index = Vec::with_capacity(count);
        let mut i = 4;
        for _ in 0..count {
            let kl = u32::from_le_bytes(idx[i..i+4].try_into()?) as usize; i += 4;
            let key = Bytes::copy_from_slice(&idx[i..i+kl]); i += kl;
            let off = u64::from_le_bytes(idx[i..i+8].try_into()?); i += 8;
            index.push((key, off));
        }
        Ok(Self { raw, index })
    }

    /// 点查:对 index 二分,命中则读 data 区那条 entry 的 value。O(log n)。
    pub fn get(&self, key: &[u8]) -> Option<Bytes> {
        let pos = self.index.binary_search_by(|(k, _)| k.as_ref().cmp(key)).ok()?;
        let off = self.index[pos].1 as usize;
        let d = &self.raw;
        let kl = u32::from_le_bytes(d[off..off+4].try_into().ok()?) as usize;
        let vstart = off + 4 + kl;
        let vl = u32::from_le_bytes(d[vstart..vstart+4].try_into().ok()?) as usize;
        Some(Bytes::copy_from_slice(&d[vstart+4..vstart+4+vl]))
    }

    pub fn len(&self) -> usize { self.index.len() }
    pub fn is_empty(&self) -> bool { self.index.is_empty() }
}

🧰 Coding style：零拷贝解析 + Option 组合子 + ? 传播

  • 零拷贝解析:open 把整文件读进一个 raw: Vec<u8>,之后 index/footer 全是对 raw 的 slice(&raw[a..b]),不复制。只有 get 要把 value 交出函数边界时才 Bytes::copy_from_slice。这正是前面 Borrow/Slice 讨论的落地:解析期借用、边界处才拷贝。任意 parser(网络协议、文件格式)通用。
  • Option 组合子 + ?:get 用 binary_search_by(...).ok()? 把"没找到"短路成 None;open 用 ? 把每一步 try_into 失败向上抛。错误/缺失沿一条直路传播,不靠嵌套 if——FP 的 Result/Either monad 思想,迁移到任何有 sum type 的语言。
  • 解析失败 vs 逻辑缺失:注意区分——open 的 ? 是"文件坏了"(Err),get 的 ? 是"key 不在"(None)。两种"没有"用两种类型表达,语义不混。

🧠 System mindset：index/data 分离 · 二分 · 内存层级局部性

SstReader 体现一条贯穿所有存储/计算系统的原理:把"找在哪"(index)和"是什么"(data)分开,让查找只碰小而热的 index。

原理本课同一原理在别处
index/data 分离index 入内存,data 留盘按需读B-tree 内节点 vs 叶 · DB 二级索引 · 倒排索引
有序 → 二分 O(log n)binary_search_by任何有序数组/外存查找 · std::lower_bound
内存层级局部性小 index 热、大 data 冷CPU cache line · page cache · TLB · CDN 边缘 · GPU shared memory(HPC)

迁移收获:RocksDB 比我们多一级——两级索引(块索引定位 4KB 块 → 块内再找)。看似多一步,却在大文件上赢,因为第一级小到能整个进 cache,第二级只读一个块。这就是"分层让热数据更热"的通用直觉:CPU 的 L1/L2/L3、网络的 cache 层级、HPC 的 register/shared/global memory,全是同一招——用一个小而快的层挡住对大而慢的层的访问。你设计任何"查找密集"系统时,先问:热的那一小块能不能单独拎出来放进更快的层?

Harness 系统：格式 round-trip + 差分

为什么这样验

现在 builder + reader 都在了,可以闭环:build → open → 逐键读回。 用 BTreeMap 当 oracle(它显然有序、显然正确),把同一批 KV 喂进 SST,断言每个 key 读出的 value 一致——这把"偏移算错/字节序/二分边界"一网打尽。 再单独验边界键:比所有键小、比所有键大、中间不存在的——二分最容易在这三处出 off-by-one。

代码 2 — round-trip + 差分 harness（加进 sst.rs 的 tests）

    use std::collections::BTreeMap;
    use tempfile::tempdir;

    // 格式 round-trip:build -> open -> 全键可读 + 边界返回 None
    #[test]
    fn build_open_roundtrip() {
        let dir = tempdir().unwrap();
        let path = dir.path().join("1.sst");
        let mut b = SstBuilder::new();
        b.add(b"apple", b"red").unwrap();
        b.add(b"banana", b"yellow").unwrap();
        b.add(b"cherry", b"").unwrap(); // 空 value
        b.finish(&path).unwrap();

        let r = SstReader::open(&path).unwrap();
        assert_eq!(r.len(), 3);
        assert_eq!(r.get(b"apple").as_deref(), Some(&b"red"[..]));
        assert_eq!(r.get(b"cherry").as_deref(), Some(&b""[..]));
        assert_eq!(r.get(b"aardvark"), None); // < 所有键
        assert_eq!(r.get(b"zebra"), None);    // > 所有键
        assert_eq!(r.get(b"berry"), None);    // 中间不存在
    }

    // 差分:随机有序输入,SST.get == BTreeMap.get
    #[test]
    fn matches_btreemap_oracle() {
        let dir = tempdir().unwrap();
        let path = dir.path().join("o.sst");
        let mut model = BTreeMap::new();
        for i in 0..500u32 { model.insert(format!("key{:05}", i*7 % 1000), format!("val{}", i)); }
        let mut b = SstBuilder::new();
        for (k, v) in &model { b.add(k.as_bytes(), v.as_bytes()).unwrap(); } // BTreeMap 迭代有序 -> add 升序成立
        b.finish(&path).unwrap();
        let r = SstReader::open(&path).unwrap();
        for (k, v) in &model {
            assert_eq!(r.get(k.as_bytes()).as_deref(), Some(v.as_bytes()));
        }
        assert_eq!(r.get(b"nope"), None);
    }
