TIER 2 · PHASE K

对外范围扫描 — DBIter + scan_as_of

J 给点读装上了时间旅行(get_as_of)。Phase K 给范围读也装上:把 G2 那个"读最新、在归并层去重"的 scan,升级成一个真正对外的游标 DBIter——按 snapshot 过滤版本、按 user_key 取最新可见版本去重、跳墓碑。这正是 RocksDB 的 db_iter.cc 在做的事。K 收尾,引擎的两条读路径(点 + 范围)就都快照一致 + 墓碑感知,M2 多版本引擎达成。

K 的关键动作是"分层归位":把"去重 + 快照过滤 + 墓碑跳过"这三件懂语义的事,从底层 MergeIterator 拆出来、搬回 DBIter 层。MergeIterator 退回成纯 k 路归并器(吐所有版本),DBIter 在它之上做语义判断——这才是 RocksDB 的真实分工。G2/H2 一度把去重塞进 MergeIterator,是因为当时还没有快照;一旦要按 seq 过滤,就只能在懂 seq 的那一层做。

一个最小步

本阶段是单课:K — DBIter + scan_as_of。

  • DBIter(新模块):包 MergeIterator::new(吐所有版本),上层做 snapshot 过滤 + user_key 去重 + 墓碑跳过。
  • Engine::scan_as_of(start, end, snapshot):范围版时间旅行;scan = scan_as_of(.., self.seq)。
  • 把去重逻辑从 MergeIterator 拆回 DBIter,归并层回归纯净。
🏁 K 收官 = Tier 2 / M2 多版本引擎全部达成。 InternalKey(H)→ 写路径带 seq(H)→ 墓碑删除(H)→ 点读快照(J)→ 范围读快照(K)。整条读写路径都按版本号运作。下一里程碑 Tier 3:compaction —— 真正回收被遮蔽的旧版本与墓碑,并引入"快照安全水位"。
