TIER 3 · PHASE L

Manifest / Version / VersionEdit

Tier 2(M2 多版本引擎)收官,进入 Tier 3 — Compaction。但在归并文件之前,引擎得先有一份权威的元数据知道"哪些文件构成当前数据库"——现在它还在扫目录猜(discover_ssts,Phase C 埋的"故意空洞")。Phase L 把扫盘换成 RocksDB 的做法:一份 append-only 的 MANIFEST 日志,记一串 VersionEdit(文件集增量),折叠成 Version(活文件集);用 CURRENT 文件 + 原子 rename 扛崩溃。

本期裁决(grilling):① 扁平——不引入 level,分层完整留给 Phase M(VersionEdit 用 tag 化编码,M 加字段只是多个 tag,老格式不破)。② 共享 log 模块——把 WAL 的帧抽出来,MANIFEST 复用,torn-tail 容错白嫖。③ CURRENT 讲透——open 强制 checkpoint + 体积过阈轮换,两个触发点共用一套原子安装。④ 旧 manifest 物理回收、孤儿文件清理 → 都诚实推迟到 Phase X 文件 GC。

三节子课

  • L1 — 共享 log 模块 + VersionEdit
    把 WAL 的 crc+len+payload 帧抽成 payload-无关的 log 模块;VersionEdit 的 tag 化编解码。🧰 framing≠payload · 🧠 一切皆日志(state=fold(events)) · 🧪 编码 round-trip + 日志 torn-tail。
  • L2 — Version 折叠 + 恢复 + CURRENT 原子切换 ★
    一串 edit 折叠成活文件集;读 CURRENT→replay 重建;7 步原子安装(含 fsync 父目录)。🧰 制造-再发布 · 🧠 ★原子指针切换三粒度阶梯(CAS / Arc / rename)+ 并发无锁发布 · 🧪 元数据崩溃恢复(新 harness 系统)。
  • L3 — 接进 Engine,扫盘退役
    open 用 recover 替换 discover_ssts;flush 发 edit(SST→manifest→WAL 截断的提交点排序);next_sst→共享 next_file_number。🧠 提交点/元数据 write-ahead · 🔄 孤儿 SST 回归守卫。

累计 harness 系统

表驱动 · 模型差分 oracle · 不变量 · 故障注入 · 崩溃恢复 · 格式 round-trip · 跨持久化 round-trip · 分层查找差分 · 范围时间旅行 → + 元数据崩溃恢复(第 9 种,L2):沿原子安装序列注入崩溃,断言恢复永远落在一致状态(全旧/全新)、绝不撕裂。

🏁 Phase L 收官 = Tier 3 Compaction 的地基。 有了"加文件/删文件"的增量账本(VersionEdit)和折叠语义(Version),compaction(Phase N/O)才有办法表达"归并掉一批旧文件、产出一批新文件"这个核心动作。下一站 Phase M:分层布局,给 Version 装上 level,本期"号序=新旧"的假设到那里被正式打破。
