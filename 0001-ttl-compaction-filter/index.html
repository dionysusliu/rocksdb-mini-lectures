TTL CompactionFilter：你亲手实现过期删除

脚手架已就位。你只写一件事——过滤器的删/留判断逻辑。跑出 [PASS] 即通关。

Objective

实现一个 TTL CompactionFilter：compaction 时自动删除过期的 key。这是 mission"修改源码实现新功能"里最常用的真实生产功能（Kvrocks/Pika 的 key 过期、Flink 状态清理都靠它），且是公开 API、不用改 RocksDB 源码。

Prereqs

已有 lab/ 工作区 + 已编译的 librocksdb.dylib（前 8 个 lab 用的那个）。脚手架文件 lab/src/09_compaction_filter.cc 已生成并注册进 CMake。

背景：value 编码（脚手架已写好）

每个 value 前 8 字节存写入时间戳（unix 秒，小端），后面是 payload：

value = [8字节 write_time][payload]
         └ EncodeValue() 写入   └ DecodeWriteTime() 读出  (都已写好,直接调用)

脚手架的 main() 会：写 1000 个 key（偶数 key 写入时间 = now-500 秒 过期，奇数 key = now-10 秒 新鲜）→ flush + CompactRange 触发 filter → 验证过期 key 被删、新鲜 key 保留 → 打印 PASS/FAIL。

Steps — 你要填的 3 个 TODO

1

TODO 1 — 实现 FilterV2

在 TTLFilter 类里加这个方法。这是核心：每个 compaction 流过的 key 都会回调它，你返回 Decision 决定留还是删。

Decision FilterV2(int level, const Slice& key, ValueType value_type,
                  const Slice& existing_value, std::string* new_value,
                  std::string* skip_until) const override {
  (void)level; (void)key; (void)new_value; (void)skip_until;  // 消未用警告
  if (value_type != ValueType::kValue) return Decision::kKeep;  // 只管普通值
  uint64_t write_time = DecodeWriteTime(existing_value);
  if (now_ - write_time > ttl_seconds_) return Decision::kRemove;  // 过期→删
  return Decision::kKeep;
}
为什么先判 value_type： merge operand / wide-column 的 value 不是你的编码格式，盲解时间戳会读到垃圾。只处理 kValue 是正确性底线（回想讨论里的语义陷阱）。
2

TODO 2 — 实现 Name()

const char* Name() const override { return "TTLFilter"; }

RocksDB 要求每个 filter 有名字（日志/诊断用）。一行。

3

TODO 3 — 注册过滤器

在 main() 里 DB::Open 之前，把 filter 挂到 options：

static TTLFilter filter(kNow, kTTL);
options.compaction_filter = &filter;
为什么 static： filter 必须在整个 DB 生命周期内存活（compaction 是后台异步的）。栈变量会提前析构 → 悬垂指针。static 让它活到程序结束。生产中无状态 filter 用全局，有状态用 compaction_filter_factory 每次 compaction 建新实例。
