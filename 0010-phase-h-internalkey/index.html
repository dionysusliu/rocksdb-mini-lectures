TIER 2 · PHASE H

InternalKey — seq + type + 墓碑

Phase G 的归并靠源优先级猜新旧——compaction 把多版本混进一个 SST 后就失灵。Phase H 给每个 key 编上序列号 seq(谁新)和类型 type(Put/Delete):新旧成为 key 自带、比较器可读的信息,删除成为一种可归并的写(墓碑)。这是从"能存能读"迈向"多版本语义"的关键一步,也兑现 A1 埋下的"plain key 不够"张力。

设计主线:字节可比编码。 InternalKey = user_key ‖ be(u64::MAX − seq) ‖ [type],让裸字节 Ord 直接给出 user_key 升 / seq 降。于是 memtable(SkipMap 要 K: Ord)、SST 块二分、归并堆全用默认字节比较、零 comparator 穿线;唯一"懂 InternalKey"的逻辑收敛到读路径与归并去重(H3)。

三个最小步

STEP H1 · done

InternalKey 编码 + 字节可比

internal_key.rs:encode/user_key/seq/value_type/lookup_key。取反 seq 编码让默认 Ord 给出 user_key 升 / seq 降。纯格式,不碰存储。对标 dbformat.h。

harness:round-trip · 排序契约(a@9<a@5<a@3<b@1) · lookup_key 落最新可见
STEP H2 · done

写路径穿 seq

引擎全局 seq 计数器,每 put 自增;memtable 存 encode(uk, seq, VALUE);WAL record 带 seq+type(replay 重建版本)。get/scan 顺势改"按 user_key 取最新版本"(否则存 InternalKey 却用 plain key 查会失配)。

harness:写路径 round-trip · seq 单调 · 覆盖写取最新
STEP H3 · done

墓碑 + delete() + 归并比较器重写

delete(k) 写 TYPE_DELETION 墓碑;get 见墓碑返回 None;MergeIterator 去重从"全 key 相等"改成"user_key 相等",墓碑遮蔽旧值且自己不吐。get(k) ≡ scan(k..=k).first() 互锁两路径。

harness:墓碑可见性 · get≡scan首元素 · 删后重写复活
承接 Phase J/K. H3 的读路径用一个固定的"最大 seq"读最新。Phase J 把它参数化成可传入的 snapshot_seq → 读历史快照(时间旅行)。Phase K 把 G2 的 minimal scan 升级成快照+墓碑感知的对外范围游标。本阶段所有钥匙都是 H1 的 lookup_key(uk, snap):H3 传 MAX、J 传历史 seq。
